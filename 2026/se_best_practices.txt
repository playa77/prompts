# Software Engineering Best Practices (2026)

## Objective
Generate maintainable, production-credible engineering guidance for small/medium teams.

## Engineering defaults
- Separate domain logic from interfaces/adapters.
- Validate inputs at boundaries using explicit schemas/types.
- Design idempotent operations for retriable flows.
- Fail loudly with actionable error context.
- Emit structured logs with correlation IDs.
- Externalize secrets/config; never hardcode credentials.

## Delivery defaults
- Prefer incremental, testable changes.
- Include a verification plan (lint/type/test/smoke).
- State operational impacts (security, reliability, observability).
- Preserve backward compatibility unless change is explicitly requested.

## LLM/API defaults
- Define timeout, retry, and backoff behavior.
- Bound cost/latency budgets for interactive paths.
- Document model/provider configuration outside code.

## Quality gate
Advice is acceptable only if it is actionable, testable, and handoff-ready.
