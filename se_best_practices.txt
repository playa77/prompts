Python/LangChain engineering:

- Logic/UI separation; core supports GUI without refactor.
- Explicit names; no abbreviations.
- Type hints; mypy --strict in CI.
- Validate inputs via Pydantic; actionable errors.
- Idempotent ops for retriable actions.
- Handle errors explicitly; never swallow.
- Structured logs: JSON, severity, UTC, correlation IDs.
- --verbose/--quiet; dry-run + --yes for destructive ops.
- Timeouts on all I/O.
- Exponential backoff with jitter.
- Context managers for deterministic cleanup.
- UTF-8; handle spaces in paths.
- Secrets: .env for dev only.
- Pin deps: exact versions + lockfiles.
- Tests: unit + integration + e2e + fuzz; hermetic.
- Linting + formatting in CI; warnings = errors.
- One-command dev setup; top-tier README.
- Small modules; feature flags.
- Cache with explicit keys + invalidation rules.
- Measure before optimizing.
- Least privilege; not root.
- Sanitize untrusted input; never string-concat.
- Graceful degradation.
- Health checks, readiness/liveness, metrics.
- Full files only; no snippets.
- Self-bootstrapping CLI: venv + lockfile + re-invoke.
- pathlib, argparse/typer, src/ layout, pyproject.toml.
- asyncio for I/O; ProcessPoolExecutor for CPU.
- LangChain: LCEL; Pydantic I/O; version prompts; model via RunnableConfig + env; cache with content-hash invalidation.
- Verbose comments; version tracking.
- Explicit over implicit; surprise is a bug.
